## Apa isi playbook ini?

Panduan lengkap untuk DS/DA dari awal project sampai handoff — dirancang supaya junior bisa pakai sendiri tanpa harus selalu tanya senior.

| Topik | Isi |
|---|---|
| **Project** | Question framework, tipe project, alur DE & DS, cara mulai |
| **Panduan** | Penamaan, standar notebook, Python/.py, config & kredensial, Git |
| **Analisis** | Data Quality, Testing, Experiment Tracking, Model Evaluation, Versioning |
| **Delivery** | Insight & Rekomendasi, Dashboard Readiness, Error & Logging, Pipeline Maturity, Handoff ke DE |
| **Tim** | Kolaborasi tim, pola kerja sequential vs paralel, cara review junior |
| **Submit** | Definition of Done & Checklist per tahap |

## Cara pakai

Buka link di atas — tidak perlu instalasi apapun. Tersedia dalam **light mode & dark mode**.

Atau buka lokal: clone repo ini, buka `index.html` langsung di browser.

---

## Usul perubahan

Ada konten yang kurang tepat, ketinggalan, atau bisa diperjelas? Ada dua cara:

1. **Template proposal** — buka halaman *Usul Perubahan* di dalam playbook, isi template, kirim ke owner
2. **Langsung via Git** — buat branch `propose/[deskripsi]`, edit `index.html`, push, ping owner

Detail proses ada di halaman **Kontribusi → Usul Perubahan** di dalam playbook.
